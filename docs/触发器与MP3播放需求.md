# 触发器系统与 MP3 文件播放支持需求文档

## 1. 背景

当前应用需要新增一种可扩展的“触发器”能力，用于根据用户输入文本识别意图，并在识别到指定意图后执行对应动作。

首个明确场景是：用户通过自然语言表达“播放某个音频”的意图，系统经过大模型意图检测后，将固定 JSON 识别结果发送给调度器，由调度器执行对应的 MP3 文件播放任务。

该能力不应只为 MP3 播放定制，而应设计为通用触发机制，未来可以扩展到运行脚本、打开文件、调用命令、控制应用状态等更多功能。

## 2. 目标

1. 支持 MP3 文件播放。
2. 在偏好设置侧边栏中新增“触发器”配置项。
3. 支持用户创建多个 TriggerDefinition。
4. 每个 TriggerDefinition 包含“输入”和“触发”两个核心内容。
5. 用户输入文本后，先交由大模型进行意图检测。
6. 如果大模型判断用户输入匹配某个触发器意图，则输出固定 JSON。
7. 调度器检测到合法 JSON 识别结果后执行对应动作。
8. 支持终止功能，例如“暂停”“终止”“停止”等文本可以中断已触发的状态。
9. 触发器系统需要为未来功能扩展预留能力。

## 3. 偏好设置 UI 需求

在偏好设置侧边栏中新增一个配置项，名称为：

```text
触发器
```

“触发器”配置项的布局风格应与现有“自动化”布局保持一致，包括但不限于：

1. 显示触发器列表。
2. 支持添加、删除、编辑多个触发器。
3. 选中某个触发器后，在详情区域编辑该触发器内容。
4. 每个触发器应至少包含以下字段：
   - 触发器名称
   - 是否启用
   - 输入配置
   - 触发配置
   - 终止配置
   - 当前状态展示

## 4. 触发器配置结构

每个触发器配置项代表一个可识别、可执行、可终止的触发定义。

为避免配置与运行状态混淆，命名约定如下：

```text
TriggerDefinition：触发器配置定义，来自用户在偏好设置中的配置。
TriggerRun：一次触发执行实例，用于记录某次执行的开始、结束、结果和错误。
TriggerRuntimeState：触发器当前运行状态摘要，用于 UI 展示和调度器判断。
```

示例触发器：

```text
名称：MP3 播放识别
输入：识别输入的意图是否是要播放 XXX 音频
触发：播放通过文件选择器选择并授权的 XXX.mp3
终止：当用户表达暂停、终止、停止播放等意图时，停止当前音频播放
```

触发器字段建议如下：

| 字段 | 说明 |
| --- | --- |
| 触发器名称 | 用于在 UI 中识别该触发器，例如“MP3 播放识别”。 |
| 启用状态 | 控制该触发器是否参与意图检测。 |
| 输入描述 | 用户自定义的自然语言说明，用于告诉大模型需要识别什么意图。 |
| 触发关键词 | 系统自动生成的唯一关键词，大模型识别成功后通过 JSON 输出给调度器，例如 `TRIGGER_PLAY_MP3_001`。 |
| 触发动作 | 调度器收到合法 JSON 识别结果后执行的动作。 |
| 终止输入描述 | 用于识别用户是否想要终止当前 TriggerRuntimeState。 |
| 终止关键词 | 系统自动生成的唯一终止关键词，大模型识别到终止意图后通过 JSON 输出给调度器，例如 `TRIGGER_STOP_MP3_001`。 |
| 当前运行状态 | 未触发、已触发、正在执行、已终止、执行失败等。 |

## 5. 输入与意图检测流程

用户输入任意文本后，系统不应直接执行动作，而应先经过大模型进行意图检测。

流程如下：

1. 用户输入文本。
2. 系统读取所有已启用的触发器。
3. 将用户输入和触发器的“输入描述”交给大模型。
4. 大模型判断用户输入是否匹配某个触发器；如多个触发器可能匹配，必须按 UI 列表顺序只返回第一个最佳匹配。
5. 如果匹配，大模型必须输出固定 JSON 结构。
6. 调度器接收到 JSON 识别结果。
7. 调度器根据 `triggerId`、`type` 和 `keyword` 查找对应动作。
8. 执行动作。
9. 如果未匹配任何触发器，则继续走普通聊天流程。

示例：

```text
用户输入：
播放那首 XXX 音频

触发器输入描述：
识别输入的意图是要播放 XXX 音频

大模型输出：
{
  "matched": true,
  "triggerId": "mp3_play_recognition",
  "type": "start",
  "keyword": "TRIGGER_PLAY_MP3_001"
}

调度器动作：
播放通过文件选择器选择并授权的 XXX.mp3
```

## 6. 大模型输出约束

为保证调度器稳定工作，大模型输出必须是结构化且可预测的。

要求：

1. 识别到触发意图时，只输出固定 JSON。
2. 识别到终止意图时，只输出固定 JSON。
3. 未识别到任何触发器时，输出 `{"matched": false}`。
4. 不允许大模型直接输出可执行命令。
5. 不允许大模型自由生成文件路径、Shell 命令或系统操作。
6. 所有可执行动作必须来自用户在偏好设置中明确配置的触发动作。
7. 调度器不能只依赖裸关键词执行动作，必须同时校验 `triggerId`、`type` 和 `keyword`。
8. JSON 解析失败、字段缺失、`triggerId` 对不上、`type` 不合法、`keyword` 对不上、模型输出多余文本等情况，全部视为未匹配，并继续走普通聊天流程。
9. 意图检测 prompt 必须要求模型只返回一个 JSON 对象，并且只返回 UI 列表顺序中的第一个最佳匹配。

建议输出形式：

```json
{
  "matched": true,
  "triggerId": "mp3_play_recognition",
  "keyword": "TRIGGER_PLAY_MP3_001",
  "type": "start"
}
```

终止示例：

```json
{
  "matched": true,
  "triggerId": "mp3_play_recognition",
  "keyword": "TRIGGER_STOP_MP3_001",
  "type": "stop"
}
```

无匹配示例：

```json
{
  "matched": false
}
```

## 7. 调度器需求

调度器负责接收大模型输出的 JSON 识别结果，并执行对应动作。

调度器不直接理解自然语言，只处理结构化识别结果。

调度器职责：

1. 维护 `TriggerDefinition`、关键词与动作的映射关系。
2. 接收大模型识别结果。
3. 校验 `triggerId`、`type` 和 `keyword` 是否合法。
4. 检查触发器是否启用。
5. 执行动作。
6. 维护 `TriggerRun` 和 `TriggerRuntimeState`。
7. 支持终止动作。
8. P0 维护 UI 当前状态和错误信息；P1 再记录持久化历史日志。

调度器必须保证：

1. 只有已配置且通过 `triggerId`、`type`、`keyword` 联合校验的识别结果可以触发动作。
2. 未知关键词不执行任何操作。
3. 禁用的触发器不执行。
4. 执行失败时不影响其他触发器。
5. 同一个触发器重复触发时，首期固定使用“先终止再重新执行”。

重复触发策略后续可提供配置：

```text
忽略重复触发
重新执行
先终止再重新执行
允许并行执行
```

MP3 播放场景首期固定使用：

```text
先终止再重新执行
```

## 8. MP3 播放需求

首期需要支持播放用户通过文件选择器选择并授权的 MP3 文件。

MP3 播放动作配置字段：

| 字段 | 说明 |
| --- | --- |
| 动作类型 | 播放音频 |
| 音频文件 | 通过文件选择器选择本地 `.mp3` 文件 |
| 文件访问授权 | 保存 security-scoped bookmark |
| 播放方式 | 从头播放 |
| 触发时已有播放任务 | 停止旧任务后播放新任务 |

MP3 播放行为要求：

1. 支持通过文件选择器选择 `.mp3` 文件。
2. 选择文件后保存 security-scoped bookmark，确保沙盒环境下后续仍可访问。
3. 文件不存在时提示错误。
4. 文件不可读或授权失效时提示错误。
5. 播放过程中应维护运行状态。
6. 支持停止播放。
7. “暂停”“停止”“终止”等用户表达首期都按终止处理，即停止当前播放。
8. 多个音频触发器同时触发时，按“先终止再重新执行”处理。
9. P0 不支持音量配置和单曲循环。

首期最低要求：

1. 播放指定 MP3。
2. 停止当前 MP3。
3. 播放失败时展示错误状态。
4. 通过 security-scoped bookmark 访问用户选择的 MP3。

## 9. 终止功能需求

触发器必须支持终止能力。

终止不是只针对 MP3，而是触发器系统的通用能力。

终止流程：

1. 用户输入文本，例如：
   - 暂停
   - 停止
   - 终止播放
   - 不要播放了
   - 取消刚才那个操作
2. 系统将用户输入交给大模型进行终止意图识别。
3. 大模型输出对应触发器的终止 JSON。
4. 调度器接收终止 JSON。
5. 调度器查找当前正在运行的 TriggerRuntimeState。
6. 执行该触发器配置的终止动作。
7. 更新状态为“已终止”。

终止配置建议包括：

| 字段 | 说明 |
| --- | --- |
| 终止输入描述 | 识别用户是否想停止、暂停、取消当前 TriggerRuntimeState。 |
| 终止动作 | 停止音频、终止进程、取消任务、执行自定义命令。 |
| 终止范围 | 仅终止当前触发器、终止所有正在运行的触发器。 |

MP3 场景中：

```text
触发动作：播放 XXX.mp3
终止动作：停止当前音频播放
暂停、停止、终止等表达均视为终止。
```

## 10. 未来扩展能力

触发器系统不应只绑定 MP3 播放，应支持后续增加更多动作类型。

建议动作类型设计为可扩展枚举：

```text
播放音频
运行文件
运行 Shell 命令
打开应用
打开 URL
调用自动化任务
发送系统通知
控制角色状态
调用插件能力
执行自定义脚本
```

首期可以只实现：

```text
播放音频
运行文件 / 命令的配置入口可预留，但可暂不开放或标记为后续支持
```

## 11. 安全需求

由于触发器可能执行命令或运行文件，必须增加安全限制。

要求：

1. 大模型不能直接生成命令执行。
2. 所有动作必须由用户在偏好设置中显式配置。
3. 执行动作前必须校验 `triggerId`、`type` 和 `keyword`。
4. 对文件路径进行存在性、权限和 security-scoped bookmark 有效性校验。
5. 对未来 Shell 命令执行增加确认或白名单机制。
6. 可在 UI 中标记高风险动作。
7. 触发器配置应持久化保存，但敏感信息不得明文暴露在日志中。
8. 调度器日志不能记录用户隐私文本的完整内容，或需要提供开关。

## 12. 状态与日志

P0 每个触发器应维护 UI 当前状态：

```text
未启用
待触发
识别中
已触发
执行中
执行成功
执行失败
已终止
```

P1 持久化历史日志建议记录：

```text
触发器名称
TriggerDefinition ID
TriggerRun ID
触发时间
用户输入摘要
大模型识别结果
输出关键词
执行动作类型
执行结果
失败原因
终止时间
```

## 13. 示例配置

```text
触发器名称：
MP3 播放识别

启用：
是

输入：
识别用户输入的意图是否是要播放 XXX 音频。用户可能会说“播放 XXX”“我想听 XXX”“开始放那个 XXX 音频”等。

触发关键词：
TRIGGER_PLAY_MP3_001

触发动作：
播放音频

音频文件：
通过文件选择器选择 XXX.mp3，并保存 security-scoped bookmark

终止输入：
识别用户是否想暂停、停止、终止、取消当前音频播放。

终止关键词：
TRIGGER_STOP_MP3_001

终止动作：
停止当前音频播放
```

## 14. 验收标准

1. 偏好设置侧边栏中出现“触发器”配置项。
2. 可以新增、删除、编辑多个触发器。
3. 每个触发器可以配置输入描述和触发动作。
4. 可以通过文件选择器配置 MP3 文件，并保存访问授权。
5. 用户输入匹配播放意图后，大模型输出固定 JSON：`matched + triggerId + type + keyword`。
6. 调度器联合校验 `triggerId`、`type`、`keyword` 后播放对应 MP3。
7. 用户输入“停止”“暂停”“终止播放”等文本后，均按终止处理并停止当前播放。
8. 禁用触发器后，不再响应对应输入。
9. MP3 文件不存在时，不崩溃，并展示错误。
10. 未匹配任何触发器时，不执行触发动作，并继续走普通聊天流程。
11. 多个触发器存在时，只执行匹配到的触发器；若多个触发器同时匹配，首期按 UI 列表顺序选择第一个最佳匹配，其余不执行。
12. 大模型不会直接控制系统执行任意命令。
13. 大模型输出 JSON 解析失败、字段缺失或校验不通过时，不执行触发动作，并继续普通聊天。
14. P0 不支持音量配置和单曲循环。

## 15. 非目标

首期不要求实现以下能力：

1. 不要求支持所有音频格式。
2. 不要求支持音频播放列表。
3. 不要求支持在线音频 URL。
4. 不要求支持复杂多轮对话确认。
5. 不要求大模型自动选择本地文件。
6. 不要求大模型直接生成或执行系统命令。
7. 不要求实现完整插件市场式的动作扩展系统。

## 16. 建议优先级

### P0

```text
触发器 UI
触发器持久化
大模型独立结构化意图检测
固定 JSON 输出
调度器 JSON 识别结果匹配
MP3 播放
MP3 停止
基础错误处理
未匹配后继续普通聊天
```

### P1

```text
触发器日志
重复触发策略
暂停 / 继续播放
多个触发器状态展示
终止所有触发器
```

### P2

```text
运行文件
运行命令
动作权限管理
更多音频格式
触发器导入导出
插件化动作扩展
```
