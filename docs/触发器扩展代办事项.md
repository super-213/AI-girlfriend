# 触发器扩展代办事项

## 背景

当前触发器系统已实现通用识别与调度框架，但 P0 可执行动作只支持本地 MP3 播放和停止。

以下事项用于记录后续需要扩展的触发器动作能力。

## 待办

- [ ] 支持触发运行 Shell 命令。
- [ ] 支持触发打开 URL。
- [ ] 支持触发打开 App。
- [ ] 支持触发运行本地文件。
- [ ] 支持触发调用自动化任务。
- [ ] 支持触发执行脚本。
- [ ] 支持触发控制角色状态。

## 实现注意事项

- 所有动作必须来自用户在偏好设置中显式配置，不能由模型直接生成可执行内容。
- 调度器必须继续校验 `triggerId`、`type` 和 `keyword` 后再执行动作。
- Shell 命令、脚本、本地文件运行等高风险动作需要增加确认、白名单或权限提示。
- 文件类动作需要保存并校验 security-scoped bookmark。
- URL、App、自动化任务、角色状态控制应分别定义明确的动作配置结构，避免复用裸字符串执行。
